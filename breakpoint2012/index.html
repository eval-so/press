Breakpoint Evaluation


Public Code Execution API
What is

Breakpoint?

Code evaluation "in ze cloud!"

  • Provides a Public API for Code Compilation and Execution.
  • Uses SELinux-powered Sandboxes
  • Is written in Scala
  • Is entirely Open Source (GPLv2+)

An open-source codepad.com alternative that rocks.

  • Scales. Scales well.
  • Very simple sign-up process.
  • Officially supported API clients in Ruby and Scala.
  • Evaluates/compiles many different languages.
  • Ruby
  • Python
  • Perl
  • Clojure
  • Scala
  • Haskell
  • Io
  • ...the list goes on.

How does it work?

The workflow goes like this.

  • A request hits the API (/evaluate).
  • The "frontend" has access to a database of available compilation nodes, and picks the lowest-current-load node that can evaluate the selected language.
  • Every node has a daemon, "BCS" (Breakpoint Compilation System), listening on it. It is powered by Akka actors.
  • The frontend forms a message containing information about the code to evaluate, and sends it off to the node it chose.
  • The node sets up a sandbox environment, copies any needed files into it, and runs the compilation/evaluation within the sandbox.
  • When it's done, BCS sends back a message to the frontend, which contains the result.

What does a request look like?

{
  "api_key": "YOUR_API_KEY",
  "language": "ruby-1.9.3p0",
  "code": "puts 'Hello world!'"
}

Yeah. It's that simple.

Not to worry - it can do much, much more.

Some noteworthy optional features:
  • Screenshots of Xorg output from evaluated code.
  • Support for multiple architectures (i686, x86_64*, sparc64, ppc64, etc).
  • Optionally, send STDIN to a process.

* = default

So - why should you care? Well..

  • Find some random code on Reddit? Don't test it on your own box!
  • Run a site that deals with code (like a pastebin?) - This adds a nice touch!
  • IRC/Campfire bots, CI testing, etc.
